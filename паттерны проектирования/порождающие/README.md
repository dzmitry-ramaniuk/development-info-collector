# Порождающие паттерны

Порождающие паттерны (Creational Patterns) — это паттерны проектирования, которые решают задачи создания объектов. Они помогают сделать систему независимой от способа создания, композиции и представления объектов.

## 🎯 Как использовать

### Для начинающих
Начните с паттернов Singleton и Builder — они наиболее распространены и часто встречаются в коде. После этого переходите к Factory Method, который является основой многих архитектурных решений.

### Для подготовки к собеседованиям
Обязательно знайте:
- **Singleton**: Различные способы реализации (Eager, Lazy, Double-Checked Locking, Enum)
- **Factory Method vs Abstract Factory**: В чем разница и когда использовать каждый
- **Builder**: Когда его применять и как реализовать fluent interface
- Примеры использования этих паттернов в JDK (например, `Calendar.getInstance()`, `StringBuilder`)

### Для опытных разработчиков
Изучите нюансы:
- Проблемы Singleton в многопоточной среде и их решения
- Использование порождающих паттернов в DI-контейнерах (Spring)
- Комбинирование паттернов (например, Abstract Factory + Singleton)
- Альтернативы классическим реализациям в современной Java

## 💡 Рекомендации

- **Singleton**: В современных приложениях предпочитайте использовать DI-контейнеры вместо ручной реализации Singleton
- **Factory Method**: Используйте статические фабричные методы вместо конструкторов для более выразительного API
- **Builder**: Применяйте для объектов с большим количеством необязательных параметров
- **Prototype**: Помните о проблемах при клонировании объектов со ссылками на изменяемые поля

## ⚠️ Важные замечания

> **Singleton и тестирование**: Singleton может усложнить тестирование, так как создает глобальное состояние. Рассмотрите использование dependency injection.

> **Builder и immutability**: Builder отлично подходит для создания неизменяемых объектов, так как позволяет установить все поля перед созданием.

> **Factory vs Constructor**: Фабричные методы имеют преимущества перед конструкторами: они могут иметь понятные имена, могут возвращать подтипы, могут кешировать экземпляры.
