## Роль SNS в архитектуре

**SNS** — managed pub/sub сервис для публикации событий сразу нескольким независимым потребителям.

## Типы подписчиков

- SQS
- Lambda
- HTTP(S) endpoint
- Email/SMS/mobile push

## Паттерн fan-out

Типовая схема: `Producer -> SNS Topic -> N подписчиков`.

Преимущества:
- слабая связность сервисов;
- независимое масштабирование потребителей;
- простой способ добавить нового подписчика без изменений у продюсера.

## Надёжность и безопасность

- Для критичных систем подписывайте SQS, а не прямые HTTP endpoints.
- Настраивайте политики topic’ов и шифрование.
- Для повторной обработки сообщений используйте DLQ на стороне подписчиков.
