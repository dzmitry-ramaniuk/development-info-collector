## Когда выбирать EC2

**EC2** подходит, когда нужен полный контроль над ОС, агентами мониторинга, сетевыми параметрами и окружением рантайма.

Типовые кейсы:
- long-running backend-сервисы;
- legacy-приложения с особыми системными зависимостями;
- self-hosted инфраструктурные компоненты.

## Ключевые компоненты

- **AMI** — шаблон образа инстанса.
- **Instance Type** — CPU/RAM/сеть/диск.
- **EBS** — блочные диски для данных.
- **Security Group** — stateful firewall.
- **IAM Role** — доступ инстанса к AWS API без статических ключей.

## High Availability и масштабирование

- Размещайте инстансы в нескольких AZ.
- Используйте **Auto Scaling Group (ASG)**.
- Ставьте перед ASG **Application Load Balancer (ALB)**.
- Для stateful-компонентов выносите состояние в managed-сервисы (RDS, ElastiCache, S3).

## Безопасность

- Отключайте прямой SSH из интернета, используйте SSM Session Manager.
- Применяйте принцип **least privilege** в IAM Role.
- Регулярно патчите ОС и обновляйте AMI.
- Шифруйте EBS-диски и бэкапы snapshot’ов.

## Практический baseline для production

1. ASG минимум на 2 AZ.
2. ALB + health checks.
3. CloudWatch-метрики и алерты (CPU, память через агент, 5xx, latency).
4. Централизованные логи в CloudWatch Logs/OpenSearch.
5. Blue/green или rolling deployment через CI/CD.
