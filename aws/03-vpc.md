## Базовые концепции

**VPC** — изолированная виртуальная сеть в AWS.

Основные элементы:
- CIDR-блок VPC;
- публичные и приватные подсети;
- таблицы маршрутизации;
- Internet Gateway и NAT Gateway.

## Подсети и выход в интернет

- **Публичная подсеть**: маршрут в Internet Gateway.
- **Приватная подсеть**: обычно выход через NAT Gateway.
- Базы данных и внутренние сервисы — в приватных подсетях.

## Security Groups и NACL

- **Security Group**: stateful, применяется к ENI/инстансам.
- **NACL**: stateless, применяется к подсети.

Рекомендация: первично ограничивайте доступ через Security Groups, NACL используйте как дополнительный слой.

## Связь между сервисами

- **VPC Endpoints** для приватного доступа к S3/DynamoDB и другим сервисам.
- **VPC Peering / Transit Gateway** для связи между VPC.
- **PrivateLink** для приватной публикации сервисов.

## Практический baseline для production

1. Отдельные VPC для `dev/stage/prod`.
2. Минимум 2 AZ.
3. Приватные подсети для приложений и БД.
4. Явные egress/ingress правила.
5. Flow Logs для анализа сетевой активности.
