## Зачем нужен EventBridge

**EventBridge** маршрутизирует события от AWS-сервисов и ваших приложений к целевым обработчикам по декларативным правилам.

## Event bus и правила маршрутизации

- События публикуются в event bus.
- Правила фильтруют события по полям (source, detail-type, detail).
- Каждое правило отправляет событие в target (Lambda, SQS, Step Functions и др.).

## Интеграции и сценарии

- Планировщик (cron/rate) для периодических задач.
- Реакция на события AWS (например, изменения состояния ресурсов).
- Интеграция между bounded contexts через доменные события.

## Контракты событий

- Версионируйте event schema.
- Не ломайте обратную совместимость.
- Добавляйте correlation id и метаданные трассировки.

## Практический baseline для production

1. Отдельные bus/правила по доменам.
2. Retry policy и DLQ на target’ах.
3. Мониторинг failed invocations.
4. Схемы событий задокументированы.
5. Минимальные привилегии IAM на публикацию/чтение.
