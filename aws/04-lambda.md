## Когда выбирать Lambda

**Lambda** подходит для event-driven и bursty-нагрузки, когда не хочется управлять серверами.

Кейсы:
- API через API Gateway;
- обработка событий из S3/SQS/EventBridge;
- cron-задачи и интеграционные пайплайны.

## Модель выполнения

- Функция запускается по событию.
- Контейнер выполнения может переиспользоваться (warm start).
- При росте нагрузки масштабирование происходит автоматически.

Ограничения:
- лимит времени выполнения;
- память и ephemeral storage ограничены;
- cold start влияет на p95/p99 latency.

## Интеграции

- API Gateway + Lambda для HTTP API.
- SQS + Lambda для асинхронной обработки.
- EventBridge + Lambda для событийной архитектуры.

## Производительность и стоимость

- Стоимость зависит от числа вызовов и времени выполнения.
- Оптимизируйте размер пакета и инициализацию.
- Выбирайте память по бенчмаркам (больше памяти = больше CPU).

## Практический baseline для production

1. Dead-letter или on-failure destination.
2. Idempotency для обработчиков.
3. Structured logging + correlation id.
4. Concurrency limits и алерты.
5. IaC для всех функций и триггеров.
