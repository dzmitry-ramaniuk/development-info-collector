# Базы данных

Материалы по реляционным и нереляционным СУБД, оптимизации запросов и проектированию баз данных.

## 📚 Содержание

### [PostgreSQL](postgresql/README.md)

Полное руководство по PostgreSQL — мощной объектно-реляционной СУБД:

- **Архитектура**: Внутреннее устройство, процессы, хранение данных
- **Типы данных**: Встроенные и пользовательские типы, JSON, массивы
- **Индексация**: B-tree, Hash, GiST, GIN, BRIN, стратегии индексирования
- **Производительность**: Планировщик запросов, оптимизация, EXPLAIN/ANALYZE
- **Транзакции**: MVCC, уровни изоляции, блокировки, консистентность
- **Эксплуатация**: Backup, репликация, масштабирование, мониторинг
- **Партиционирование**: Декларативное партиционирование, шардинг, стратегии

→ [Перейти к материалам по PostgreSQL](postgresql/README.md)

## 🧭 Рекомендуемые маршруты по разделу

- **Если база знаний по SQL ещё неустойчива**: начните с архитектуры и типов данных, затем переходите к индексам, транзакциям и оптимизации
- **Если готовитесь к backend-собеседованию**: особое внимание уделите темам [индексации](postgresql/03-indexing.md), [транзакций](postgresql/05-transactions.md) и [производительности](postgresql/04-query-performance.md)
- **Если хотите понимать эксплуатацию БД**: после основ изучайте [эксплуатацию и резервное копирование](postgresql/06-operations.md), затем переходите к [шпаргалке по `psql`](postgresql/08-psql-cheatsheet.md) и [партиционированию](postgresql/09-partitioning-sharding.md)

## 💡 Рекомендации по изучению

1. **Начните с основ** — реляционная модель, нормальные формы, SQL
2. **Изучите PostgreSQL** — одна из самых мощных и популярных СУБД
3. **Практикуйтесь** — запускайте локальные инстансы, пишите запросы
4. **Анализируйте планы** — используйте EXPLAIN для понимания производительности
5. **Изучайте индексы** — правильная индексация критична для производительности
