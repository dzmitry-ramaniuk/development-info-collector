## 🎯 Что такое ActiveMQ

Apache ActiveMQ — это популярный брокер сообщений с открытым исходным кодом, написанный на Java. Он реализует спецификацию JMS (Java Message Service) и поддерживает множество протоколов и языков программирования.

### Ключевые возможности

- **Поддержка стандартов**: Полная реализация JMS 1.1 и JMS 2.0
- **Множество протоколов**: OpenWire, STOMP, AMQP, MQTT, WebSocket
- **Гибкая маршрутизация**: Queues, Topics, Virtual Destinations, Composite Destinations
- **Высокая доступность**: Master/Slave конфигурации, Network of Brokers
- **Персистентность**: Поддержка различных хранилищ (KahaDB, JDBC, LevelDB)
- **Расширяемость**: Плагины, interceptors, custom destinations

### Основные сценарии использования

- Асинхронное взаимодействие между приложениями
- Интеграция разнородных систем
- Реализация паттерна publish-subscribe
- Обработка сообщений с гарантиями доставки
- Построение event-driven архитектур

## 🛠️ Когда ActiveMQ особенно полезен

- Когда проект уже использует **JMS** и нужна предсказуемая интеграция с Java-стеком.
- Когда важнее классический **message broker** для команд, задач и интеграции сервисов, чем стриминговая платформа.
- Когда нужно быстро объяснить различия между **Queue**, **Topic**, durable subscriptions и транзакционной обработкой сообщений.

## ⚠️ На что обратить внимание

> **Не путайте ActiveMQ и Kafka**: ActiveMQ чаще выбирают как JMS-брокер для интеграции приложений, а Kafka — как платформу для event streaming и повторного чтения событий.

> **Изучайте гарантии доставки вместе с ack/transaction model**: без этого легко запомнить термины, но не понять, как реально достигается надёжность.
