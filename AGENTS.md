# Инструкции для AI агентов

## О проекте

`development-info-collector` — образовательный репозиторий с структурированными учебными материалами на русском языке по технологиям разработки. Содержит конспекты, чек-листы и вопросы для подготовки к техническим собеседованиям.

## Критически важные правила

### 1. Нумерация файлов

- **Обязательный формат**: Все тематические файлы должны иметь префикс `NN-` (две цифры и дефис)
  - Примеры: `01-java-core.md`, `02-multithreading.md`, `03-indexing.md`
- **Исключения**: НЕ нумеруются только:
  - `README.md` — обзорные файлы
  - `.gitkeep` — служебные файлы
- **При добавлении файла**: Подберите следующий свободный номер в соответствующем каталоге
- **При вставке между файлами**: Перенумеруйте существующие файлы и обновите все ссылки

### 2. Обновление ссылок

- **Всегда обязательно**: После переименования файла найдите и обновите ВСЕ ссылки на него
- **Где искать ссылки**:
  - В `README.md` файлах всех уровней
  - В других тематических файлах
  - В перекрёстных ссылках между разделами
- **Используйте команду**: `grep -r "старое-имя-файла.md" .` для поиска всех ссылок

### 3. Структура каталогов

```

├── README.md                 # Главная страница
├── AGENTS.md                 # Этот файл

## Алгоритмы работы

### Добавление нового файла

1. **Определите каталог**: Выберите правильный тематический каталог
2. **Определите номер**: 
   - Посмотрите существующие номера в каталоге
   - Выберите следующий свободный номер
3. **Создайте файл**: Используйте формат `NN-описательное-имя.md`
4. **Обновите README**: Добавьте ссылку на новый файл в соответствующий `README.md`
5. **Проверьте навигацию**: Убедитесь, что файл доступен через главный `README.md`

### Переименование файла

1. **Переименуйте файл**: Сохраняя или изменяя номер
2. **Найдите все ссылки**: `grep -r "старое-имя.md" .`
3. **Обновите каждую ссылку**: В найденных файлах замените старое имя на новое
4. **Проверьте README**: Особенно внимательно проверьте все `README.md` файлы
5. **Тестовая проверка**: Убедитесь, что навигация работает

### Вставка файла между существующими

**Пример**: Нужно добавить материал между `01-topic.md` и `02-another.md`

1. **Перенумеруйте существующие**:
   - `02-another.md` → `03-another.md`
   - `03-third.md` → `04-third.md`
   - И т.д. для всех последующих файлов
2. **Создайте новый**: `02-new-topic.md`
3. **Обновите ВСЕ ссылки**: На переименованные файлы
4. **Обновите README**: С новым порядком файлов

### Создание нового раздела

1. **Создайте каталог**: Если нужен новый тематический каталог
2. **Создайте README.md**: С обзором раздела и списком материалов
3. **Добавьте файлы**: С правильной нумерацией
4. **Обновите главный README**: Добавьте ссылку на новый раздел
5. **Если каталог пустой**: Добавьте `.gitkeep`

## Стиль и форматирование

### Структура документа

```markdown
# Заголовок темы

## Содержание

1. [Введение](#введение)
2. [Основные концепции](#основные-концепции)
   - [Подраздел 1](#подраздел-1)
   - [Подраздел 2](#подраздел-2)
3. [Практические примеры](#практические-примеры)

## Введение
Краткое описание темы и её значимости.

## Основные концепции
### Подраздел 1
Детальное описание с примерами.

### Подраздел 2
Ещё одна важная концепция.

## Практические примеры
```java
// Примеры кода
```

## Практические упражнения
1. Задание 1
2. Задание 2

## Вопросы на собеседовании
1. **Вопрос?**
   *Ответ:* Детальный ответ с пояснениями.
```

### Правила оформления

- **Заголовки**: Используйте `#`, `##`, `###` по убыванию важности
- **Оглавление**: Добавляйте `## Содержание` в файлы с 2+ разделами второго уровня
  - Размещайте сразу после заголовка первого уровня
  - Используйте нумерованный список для уровня 2 (`##`)
  - Используйте маркированный список с отступом для уровня 3 (`###`)
  - Якорные ссылки должны соответствовать заголовкам: `[Раздел](#раздел)`
- **Списки**: `-` для маркированных, `1.` для нумерованных
- **Код**: Всегда указывайте язык в блоках кода:
  ````markdown
  ```java
  public class Example {}
  ```
  ````
- **Важное**: Выделяйте цитатами с `>`
- **Термины**: Жирным при первом упоминании: `**MVCC**`
- **Ссылки**: Используйте относительные пути

### Типичные элементы контента

- **Кириллица**: Допустима в именах каталогов и файлов

## Важные замечания для AI агентов

### При работе с этим репозиторием:

1. **Всегда проверяйте нумерацию** перед добавлением файлов
2. **Добавляйте оглавление** в содержательные файлы (с 2+ разделами)
3. **Никогда не пропускайте обновление ссылок** после переименования
4. **Сохраняйте стиль** существующих материалов
5. **Проверяйте README** на каждом уровне иерархии
6. **Используйте русский язык** для всего контента
7. **Следуйте установленной структуре** документов
8. **Добавляйте практические примеры** во все материалы
9. **Включайте вопросы для самопроверки** в конце разделов

### Типичные ошибки, которых нужно избегать:

- ❌ Создание файлов без нумерации
- ❌ Забывание добавить оглавление в содержательные файлы
- ❌ Неправильные якорные ссылки в оглавлении
- ❌ Забывание обновить ссылки после переименования
- ❌ Использование абсолютных путей в ссылках
- ❌ Нарушение установленной структуры документа
- ❌ Смешивание стилей оформления
- ❌ Отсутствие примеров или вопросов
- ❌ Игнорирование обновления README файлов

## Приоритеты

При работе с репозиторием соблюдайте следующие приоритеты:

1. **Точность информации** — превыше всего

4. **Полнота материала** — обеспечивает ценность
5. **Актуальность** — поддерживает релевантность
