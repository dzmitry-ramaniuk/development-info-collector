## 💡 Рекомендации

- **Начинайте с модульного тестирования** (JUnit + Mockito) — это основа качественного кода
- **Используйте Testcontainers** для интеграционных тестов с реальными зависимостями вместо H2
- **Пишите тесты вместе с кодом** (TDD/BDD подход) для лучшего дизайна
- **Стремитесь к высокому покрытию** критичных участков кода, но не гонитесь за 100%
- **Делайте тесты читаемыми** — используйте @DisplayName, AAA паттерн, выразительные имена
- **Следуйте пирамиде тестирования**: 70% юнит-тесты, 20% интеграционные, 10% E2E
- **Избегайте хрупких тестов** — проверяйте результат, а не внутреннюю реализацию
- **Оптимизируйте производительность** — используйте slice-тесты, статические контейнеры, параллельное выполнение

## ⚠️ Важные замечания

> **H2 vs Testcontainers**: Избегайте использования H2 для интеграционных тестов. H2 имеет другой SQL-диалект и поведение по сравнению с PostgreSQL/MySQL. Используйте Testcontainers для реалистичных тестов.

> **@DirtiesContext**: Избегайте этой аннотации — она пересоздаёт весь Spring контекст, что делает тесты очень медленными. Используйте @AfterEach для очистки данных.

> **Thread.sleep()**: Никогда не используйте Thread.sleep() в тестах. Используйте Awaitility для асинхронного кода.

> **Моки**: Не мокайте всё подряд. Мокайте только внешние зависимости (БД, API, очереди). Value objects и простые объекты используйте реальные.
