# development-info-collector

`development-info-collector` — это рабочая тетрадь по изучению технологий разработки. Здесь собираются конспекты, чек-листы и
вопросы для самопроверки. Каждый раздел — самостоятельный набор материалов с перекрёстными ссылками и обновляется по мере
появления новых тем.

## Структура репозитория
- [Java](java/README.md) — подробные главы по Java Core и многопоточности с собеседовательными вопросами.

## Как пользоваться материалами
1. Начните с обзорной страницы нужной технологии и выберите интересующую главу.
2. Изучайте теорию: в главах много схем, пояснений, практических рекомендаций.
3. Проверьте себя по списку вопросов-ответов, а затем вернитесь к спорным моментам.
4. По мере освоения темы дополняйте заметками или ссылками на внешние источники — конспекты задуманы как живой документ.

## Обратная связь
Если вы обнаружили неточность или хотите предложить улучшение, заведите issue или откройте pull request с правками. Чем более
подробно описана проблема, тем быстрее мы сможем её исправить.
# Development Info Collector

Коллекция взаимосвязанных конспектов и памяток в формате Markdown для подготовки к техническим собеседованиям и систематизации знаний.

## Как устроен репозиторий

Основные тематические каталоги расположены в корне проекта:

- `java/` — материалы по языку Java, экосистеме и сопутствующим инструментам.
- `базы данных/` — заметки по реляционным и нереляционным СУБД, оптимизации запросов и проектированию.
- `тестирование/` — информация о видах тестирования, инструментах и практиках обеспечения качества.
- `очереди/` — сведения об очередях сообщений, шаблонах интеграции и сопутствующих технологиях.

Внутри каждого каталога планируется структура подкаталогов и файлов Markdown, объединённых по подтемам. Рекомендуемый формат именования файлов: `tema.md`, где `tema` отражает содержание документа.

## Как добавлять материалы

1. Выберите подходящий каталог для новой темы.
2. Создайте Markdown-файл с осмысленным названием.
3. Используйте заголовки, списки и таблицы для структурирования информации.
4. При необходимости добавляйте ссылки на внешние ресурсы.
5. До появления первых заметок храните в каталоге файл-заглушку `.gitkeep`, чтобы структура не терялась при переключении веток.

## Дополнительные рекомендации

- Поддерживайте единый стиль оформления: заголовки `#`, `##`, `###` по убыванию важности, списки для перечислений.
- Обновляйте существующие заметки при появлении новой информации.
- Используйте контроль версий Git для отслеживания изменений и совместной работы.

Добро пожаловать в пространство знаний — дополняйте и совершенствуйте материалы по мере подготовки!
