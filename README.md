# Development Info Collector

Структурированная коллекция учебных материалов на русском языке по различным технологиям разработки. Это рабочая тетрадь для изучения Java, баз данных, систем очередей и тестирования, предназначенная для подготовки к техническим собеседованиям и систематизации знаний.

## 🎯 Как использовать

### Для подготовки к собеседованиям
1. Начните с обзорной страницы нужной технологии (README.md в каждом разделе)
2. Изучайте материалы последовательно, следуя нумерации файлов
3. Используйте разделы «Вопросы для самопроверки» в каждой теме
4. Практикуйтесь с примерами кода и упражнениями

### Для систематизации знаний
- Используйте материалы как справочник по конкретным темам
- Дополняйте своими заметками и ссылками
- Возвращайтесь к сложным темам периодически для закрепления

### Для изучения новых технологий
- Следуйте рекомендованному порядку изучения в каждом разделе
- Начинайте с основ, постепенно переходя к продвинутым темам
- Практикуйтесь: запускайте примеры, экспериментируйте с кодом
- Используйте примеры из материалов как отправную точку

## 💡 Принципы оформления материалов

- **Структурированность**: Каждая тема имеет оглавление, логичную структуру разделов и последовательное изложение
- **Практичность**: Материалы содержат примеры кода, практические советы и реальные use cases
- **Актуальность**: Информация обновляется с учётом современных версий технологий и best practices
- **Самодостаточность**: Каждый раздел содержит достаточно информации для понимания темы
- **Перекрёстные ссылки**: Связанные темы соединены ссылками для построения целостной картины знаний

## 📝 Добавление материалов

Если вы хотите дополнить репозиторий:

1. Выберите подходящий каталог для новой темы
2. Создайте Markdown-файл с нумерацией (формат: `NN-название.md`)
3. Следуйте установленной структуре:
   - Заголовок первого уровня с названием темы
   - Оглавление для файлов с 2+ разделами
   - Введение с контекстом
   - Основные разделы с примерами
   - Вопросы для самопроверки
4. Обновите README в соответствующем разделе
5. Используйте относительные пути для внутренних ссылок
